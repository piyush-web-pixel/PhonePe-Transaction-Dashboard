#  PhonePe Transaction Dashboard

## Project Title

**PhonePe Transaction Dashboard**

An interactive **Power BI** dashboard designed to analyze digital payment transactions across various services, customer segments, and user behaviors. The dashboard provides actionable insights into transaction volume, payment value, user engagement, service performance, and transaction success rates, helping businesses monitor digital payment trends and make informed decisions.

---

##  Project Overview

The **PhonePe Transaction Dashboard** transforms raw transaction data into meaningful business insights through interactive visualizations and dynamic KPIs. It enables analysts to monitor payment performance, identify popular services, analyze user demographics, and evaluate transaction trends over time.

The dashboard is intended for **Business Analysts, FinTech Companies, Product Managers, Marketing Teams, and Decision Makers** who require a centralized view of digital payment performance.

---

##  Tech Stack

The dashboard was built using the following technologies:

-  **Power BI Desktop** – Dashboard creation and interactive reporting
-  **Power Query** – Data cleaning, transformation, and preprocessing
-  **DAX (Data Analysis Expressions)** – Custom measures, KPIs, and calculated metrics
-  **Data Modeling** – Table relationships for accurate reporting and filtering
-  **Interactive Visualizations** – KPI Cards, Area Charts, Donut Charts, Column Charts, Bar Charts, and Slicers
-  **File Format** – `.pbix` for development and `.pdf/.png` for presentation

## Data Source:youtube

**Dataset:** Simulated PhonePe Digital Transaction Dataset

The dataset contains transaction-level information including:

- Transaction ID
- Transaction Date
- Transaction Status
- Transaction Value
- Payment Service
- User Information
- Transaction Category
- Age Group
- Weekday / Weekend Indicator

The data enables comprehensive analysis of transaction performance, customer behavior, and service utilization.

##  Dashboard Features & Highlights

###  Business Problem

Digital payment platforms process millions of transactions daily. Monitoring transaction success rates, user activity, and service usage is essential for improving customer experience and operational efficiency.

Key business questions include:

- How many transactions were completed successfully?
- What is the total transaction value?
- Which services generate the highest transaction value?
- Which customer age group contributes the most transactions?
- Who are the top users by transaction value?
- How do transactions change throughout the year?
- How does weekday usage compare to weekend usage?

---

###  Goal of the Dashboard

The primary objective of this dashboard is to provide an interactive business intelligence solution that:

- Monitors digital payment performance
- Tracks transaction growth over time
- Identifies high-performing payment services
- Understands customer demographics and behavior
- Measures transaction success rates
- Supports data-driven decision-making

## Walkthrough of Key Visuals

###  KPI Cards

The dashboard provides high-level business metrics including:

-  Total Transactions
-  Total Transaction Value
-  Unique Users
-  Transaction Success Rate

These KPIs offer an instant overview of platform performance.

---

###  Transactions Over Time

An area chart visualizes transaction volume across different months, helping identify:

- Seasonal transaction patterns
- Monthly growth trends
- Peak transaction periods
- Changes in customer activity

---

###  Age Segment Contribution

A donut chart displays transaction distribution across customer age groups:

- Millennials
- Boomers
- Gen Z

This helps understand which demographic contributes the most to platform usage.

---

###  Service Transaction Value Analysis

A column chart compares transaction value generated by different services, such as:

- Loans
- Insurance
- Money Transfer
- Recharge & Bill Payments

This helps identify the most valuable services offered by the platform.

---

###  Top Users Analysis

A bar chart ranks the top users based on total transaction value, enabling businesses to identify high-value customers.

---

###  Weekday vs Weekend Usage

A donut chart compares transaction volume between weekdays and weekends, providing insights into customer payment behavior and platform engagement.

---

###  Transaction Status Filters

Interactive filters allow users to analyze transactions based on status, including:

- Successful
- Failed
- Insufficient Amount
- Server Error

This helps evaluate operational reliability and identify failure patterns.

---

###  Insights Panel

A dedicated insights section summarizes key findings from the dashboard, including:

- Overall transaction success rate
- Highest contributing age group
- Most valuable payment service
- Major business observations

This provides quick, data-driven recommendations for decision-makers.
